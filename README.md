# CodeTracker 🚀

**Unified Coding Performance Intelligence Platform**

CodeTracker is a centralized platform designed for educational institutions to aggregate, analyze, and track student coding performance across various competitive programming platforms. By replacing fragmented spreadsheets and manual tracking, CodeTracker provides actionable insights, automated rankings, and standardized metrics for students, faculty, HODs, and administrators.

## ✨ Key Features

- **Automated Data Aggregation**: Syncs directly with top coding platforms including **LeetCode, CodeChef, GeeksforGeeks, HackerRank, and GitHub**.
- **Real-Time Analytics**: Live dashboards displaying individual and aggregate performance metrics, active streaks, and progress trends.
- **Normalized Ranking System**: Fair, cross-platform scoring engine that generates department-level and institution-level leaderboards.
- **Role-Based Access Control**: Tailored experiences and permissions for **Students, Faculty, HODs, and Admins**.
- **Profile Verification**: Built-in workflows to authenticate platform usernames submitted by students.
- **Reports & Exports**: Generate downloadable reports for placement cells, curriculum planning, and internal reviews.

## 🚀 Getting Started

### Prerequisites
Make sure you have [Node.js](https://nodejs.org/) installed on your machine.

### Installation

1. **Clone the repository**
   ```bash
   git clone https://github.com/your-username/codetracker.git
   cd CodeTracker_Web/Main
   ```

2. **Install dependencies**
   ```bash
   npm install
   ```

3. **Run the development server**
   ```bash
   npm run dev
   ```

4. **Open in Browser**
   Navigate to `http://localhost:5173` to view the application.

## 👥 User Roles

- **Student**: Track personal progress, view cross-platform analytics, and check leaderboards.
- **Faculty**: Monitor section-wise performance, verify student profiles, and identify students needing guidance.
- **HOD**: Access department-level analytics, compare sections, and evaluate overall coding culture.
- **Admin**: Manage users, configure system settings, and oversee institution-wide analytics and governance.

## 💡 Use Cases

1. **Student Evaluation**: Supplementary metric alongside traditional academic grades.
2. **Placement Preparation**: Identify coding-ready students and track improvement trends.
3. **Department Analytics**: Justify curriculum changes or lab hour allocations based on coding trends.
4. **Institutional Reporting**: Generate metrics for accreditation and internal reviews.
